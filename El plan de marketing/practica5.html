Arquitectura cliente-servidor

Modelo de funcionamiento básico El cliente

  • El cliente puede ser un proceso ejecutándose en un dispositivo cualquiera
  • El servidor puede ser un proceso ejecutándose en un equipo (normalmente de altas prestaciones)

El cliente realiza una petición a un servidor:

  1. Origina el tráfico web vía Internet usando un protocolo (generalmente HTTPy suele hacerse mediante un navegador
  2. Si el servidor es capaz de responder a dicha petición entonces enviará una respuesta con el recurso o archivo que, generalmente, el cliente es capaz de mostrar.

Respuesta dinámica(I)

¿Qué pasa cuando abres tu muro de facebook?

  • El navegador envía una petición a los servidores de facebook: Enséñame el muro de esta persona
  • Cuando el servidor recibe esta petición activa sus bases de datos para ver quiénes son tus amigos, qué contenido han publicado recientemente, qué comentarios han recibido, etc.

Respuesta dinámica (II)

  • A continuación, decide qué contenidos son más relevantes, los prioriza y construye una página web para poder enviársela a tu navegador y que te la demuestre.
  • Y todo esto lo hace en unos pocos milisegundos; la página tarda menos en construirse que en llegar a tu navegador.

HTTP y HTML

HTTP

Hypertext Transform Protocol
  • Es el protocolo más usado habitualmente para intercambiar información entre cliente y servidor, con el fin de comunicarse
  • No es el único que existe: HTTPS, SMTP, FTP, SSH...

HTML

HyperText Markup Language
  • Respuesta más comúnmente intercambiada entre navegadores y servidores web
  • a forma de la información es un archivo que contiene:
    • Texto: la información real del documento
    • Marcas: elementos que sirven para que el navegador interprete cómo mostrar el documento
    • Enlaces o referencias a otros documentos: estos enlaces nos pueden conducir a otros documentos

Respuestas de una petición HTTP

  • Cuando un servidor recibe una solicitud HTTP responde con un mensaje de texto ASCII que tiene que estar correctamente formado
  • Códigos de texto y texto asociado:
    • Un número y un texto indicando la respuesta del servidor
    • lgunos códigos de estado o significado son

Algunos códigos de estado o significado son:

  • 200 0k: La solicitud se ha satisfecho con éxito
  • 404 Not found: El servidor no puedo encontrar nada que encajara con la petición
  • 301 Moved Permantly: Los datos solicitados se han trasladado de forma permanente
  • 302 Found: Los datos solicitados están en otra URL (redirección)
  • 500 Internal Server Error: Un error del servidor ha impedido satisfacer la solicitud
